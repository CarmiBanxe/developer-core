# /gsd-plan-phase

Планирует следующую фазу/спринт на основе текущего STATE.md и ROADMAP.md.

## Что делает
1. Читает `~/developer/.planning/STATE.md` — что завершено, что заблокировано
2. Читает `~/developer/.planning/roadmap/ROADMAP.md` — следующие приоритеты
3. Читает `~/developer/.planning/REQUIREMENTS.md` — технические ограничения
4. Вызывает **gsd-planner** с контекстом следующей фазы
5. Обновляет `~/developer/.planning/PROJECT.md` новым спринтом
6. Докладывает: что сделано, что дальше, что заблокировано

## Когда использовать
- После завершения предыдущего спринта
- Когда нужно переприоритизировать
- После разблокирования BT-задачи

## Аргументы
```
/gsd-plan-phase           # следующий спринт по roadmap
/gsd-plan-phase P0        # только P0 приоритет
```

## Выход
Обновлённый `~/developer/.planning/PROJECT.md` + сводка изменений.
